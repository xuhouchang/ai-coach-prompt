# Prompt 压力测试

---

## 这是什么？

不只要看 Prompt 长什么样，还要看它**扛不扛得住**。

真实使用场景里，信息会缺失、数据量会变、目标会调整。
压力测试就模拟这三种情况，看你的 Prompt 还稳不稳。

---

## 三个测试

| 测试 | 模拟什么 | PASS 条件 |
|------|---------|-----------|
| 缺少关键信息 | 抽掉一个关键变量 | Prompt 定义了兜底行为，或该项非关键 |
| 输入规模变化 | 1 条 → 100 条 | 输出结构不依赖固定规模 |
| 输出目标变化 | 换读者 / 换格式 / 换验收 | 目标可参数化，不需要重写整个 Prompt |

---

## 判定

- ✅ 三项全 PASS → 整体 PASS
- ❌ 任意一项 FAIL → 整体 FAIL

**这是有意严格的**——只在理想条件下好用的 Prompt，不算好 Prompt。

---

## 核心认知

> 只在完美条件下好用的 Prompt，不算好 Prompt。
> 真实任务会有信息缺失、规模变化、目标调整。
> 如果 Prompt 扛不住这些，它就不是合格的任务规格书。

FAIL 不可怕——每一次 FAIL 都告诉你：哪个槽位还需要补。
